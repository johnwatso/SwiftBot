SwiftBot 1.22.1: voice announcer reliability & faster encrypted connect

Version 1.22.1 · Build 2026062801

A focused follow-up to 1.22 that makes the voice Announcer dependable end to end — clean audio, a connection that stays up, and a much faster, more reliable encrypted (DAVE) handshake.

Voice Announcer:
- Fixed silent and distorted speech; announcements render and play through cleanly.
- Added a proper UDP keepalive so the voice link no longer goes stale between messages and stops reading after the first few.
- The green voice-activity ring now lights only while SwiftBot is actually talking, not on a timer while idle.
- A short end-of-speech marker is sent after each announcement so the next one starts without clipping.

Encrypted connect (DAVE):
- Connects in seconds instead of ~15: fixed an MLS key-package ordering bug that forced a slow reset-and-retry on every join.
- No more failed joins on some servers: SwiftBot completes the connection as soon as the secure session is ready, instead of waiting for a follow-up signal some servers never send.
- End-to-end encryption stays on throughout — these are reliability fixes, not a downgrade.

Diagnostics & tooling:
- The Announcer's Technical Logs now show second-resolution timestamps and a full, labelled trace of the connection and DAVE handshake, including how long each phase took.
- Spoken message content is kept out of the diagnostics log (it still appears in the Recent list).
- New: a Clear Cached Data button under General preferences that wipes cached channel, role, and member names from previously-connected servers. Settings and token are preserved.

Maintenance:
- Resolved build warnings and a deprecated API usage for a cleaner, future-proof build.

Upgrading from 1.22 needs no configuration changes. If you've moved SwiftBot to a new server, use General → Clear Cached Data to drop any stale server names.
