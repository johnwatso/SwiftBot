SwiftBot 1.22.1

Patch release · Build 2026062801

A focused follow-up to 1.22 that makes the voice Announcer dependable end to end — clean audio, a connection that stays up, and a much faster, more reliable encrypted (DAVE) handshake.

🔊 Voice Announcer — clean, reliable audio

  • Fixed silent and distorted speech. Announcements now render and play through cleanly, with correctly paced 20 ms audio frames.
  • The connection no longer goes stale. A proper UDP keepalive keeps the voice link warm between messages, so the Announcer keeps reading instead of quietly dropping after the first few.
  • No more phantom "speaking" indicator. The green voice-activity ring now lights only while SwiftBot is actually talking, not on a timer while idle.
  • Cleaner message handoff. A short end-of-speech marker is sent after each announcement so the next one starts without clipping.

🔐 Faster, more reliable encrypted connect (DAVE)

  • Connects in seconds, not 15. Fixed an MLS key-package ordering bug that was forcing a slow reset-and-retry on every join.
  • No more failed joins on some servers. SwiftBot now finishes connecting as soon as the secure session is ready, instead of waiting for a follow-up signal some servers never send (which previously timed out and failed).
  • End-to-end encryption stays on throughout — these are reliability fixes to the DAVE handshake, not a downgrade.

🩺 Diagnostics & tooling

  • Clear, timestamped voice logs. The Announcer's Technical Logs now show second-resolution timestamps and a full, labelled trace of the connection and DAVE handshake — including how long each phase took — so issues are easy to pinpoint.
  • Privacy-aware. Spoken message content is kept out of the diagnostics log (it still appears in the Recent list).
  • New: Clear Cached Data button. Under General preferences — wipes cached channel, role, and member names from previously-connected servers. Handy after moving SwiftBot to a different server. Your settings and token are preserved.

🧹 Maintenance

  • Resolved build warnings and a deprecated API usage for a cleaner, future-proof build.

Upgrading from 1.22 — no configuration changes required. If you've moved SwiftBot to a new server, use General → Clear Cached Data to drop any stale server names.
