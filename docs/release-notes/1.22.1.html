SwiftBot 1.22.1

Version 1.22.1 · Build 2026062801

A focused follow-up to 1.22 that makes the voice Announcer dependable end to end — clean audio, a connection that stays up, and a much faster, more reliable encrypted (DAVE) handshake.

🔊Voice Announcer

Clean audio and a connection that stays up.

  • Fixed silent and distorted speech: Announcements render and play through cleanly.
  • Stays connected: A proper UDP keepalive keeps the voice link warm between messages, so the Announcer no longer stops reading after the first few.
  • Accurate activity indicator: The green voice-activity ring lights only while SwiftBot is actually talking, not on a timer while idle.
  • Clean handoff: A short end-of-speech marker is sent after each announcement so the next one starts without clipping.

🔐Encrypted Connect (DAVE)

Faster, more reliable end-to-end-encrypted voice connections.

  • Connects in seconds, not ~15: Fixed an MLS key-package ordering bug that forced a slow reset-and-retry on every join.
  • No more failed joins on some servers: The connection completes as soon as the secure session is ready, instead of waiting for a follow-up signal some servers never send.
  • Encryption stays on: These are reliability fixes to the DAVE handshake, not a downgrade.

🩺Diagnostics & Tooling

Easier to inspect voice connections, with privacy in mind.

  • Detailed logs: Second-resolution timestamps and a full, labelled trace of the connection and DAVE handshake in the Technical Logs, including per-phase timing.
  • Privacy-aware: Spoken message content is kept out of the diagnostics log (it still appears in the Recent list).
  • Clear Cached Data: A new button under General preferences wipes cached channel, role, and member names from previously-connected servers. Settings and token are preserved.

🧹Maintenance

Housekeeping for a cleaner build.

  • Resolved build warnings and a deprecated API usage.
