SwiftBot 1.22

Version 1.22 · Build 2026062612

An upgrade for the voice announcer featuring support for high-quality local Piper TTS neural voices, alongside critical stability protections to prevent connection drop-outs and UI thread locks.

🗣️Piper TTS Integration

High-quality, local, and private neural text-to-speech support.

  • Custom System Voices: Support for custom neural voices registered by third-party macOS app extensions like "Piper - Neural TTS".
  • Focused Voice Selection: The announcer dropdown is now filtered to show only Apple Premium, Apple Enhanced, and custom Piper voices, filtering out legacy low-quality system defaults.
  • Setup Guide Popover: Added clear documentation directly inside the voice settings UI guiding users on how to obtain and configure Piper neural voices.

🛡️Announcer Stall Protection

Ensuring the bot keeps reading notifications even when audio services hang.

  • Timeout Safeguards: Implemented thread-safe timeout limits (15 seconds for TTS synthesis, 20 seconds for audio transmission).
  • Automatic Queue Recovery: If a voice operation hangs or drops out, the announcer catches the error, logs a diagnostic warning, resets its draining flag, and continues reading subsequent notifications.
  • Sendable Buffer Safety: Structured under a new thread-safe sendable container to adhere to Swift Concurrency strict compile-time checks.

🔌Speaking Indicator & UDP Pacing

Paced keepalive checks reduce packet overhead and fix speaking rings.

  • Paced UDP Keepalives: Lowered background keepalive packet frequency from 20 ms to 5 seconds.
  • Speaking Ring Fix: SwiftBot now displays the green speaking indicator ring in Discord only when it is actually reading a notification, turning off immediately when silent.
  • Network Congestion Reduction: Reducing background keepalive traffic by 250x prevents socket buffer backups, resolving connection drops after 2-5 minutes of silent operation.

⚡UI Priority Inversion Fix

Preventing speech preview queries from blocking the main thread.

  • Background Preview Synthesis: Offloaded AVSpeechSynthesizer state and preview calls to a background task.
  • Unblocked Main Actor: Eliminates thread priority inversion warnings (User-Interactive threads blocking on Default thread XPC speech daemons), ensuring UI responsiveness and preventing actor queue deadlocks.
