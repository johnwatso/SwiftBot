SwiftBot 1.22: streaming announcer, voice stability, DAVE diagnostics

Announcer/voice:
- Stream TTS synthesis straight into the Opus pipeline to cut playback latency
- Realign RTP timestamps to wall-clock after idle/keepalive gaps so resumed
audio isn't scheduled late by Discord
- Replace the fixed playback timeout with an inactivity (stall) watchdog so long
messages play in full; cancelling playback now stops the synthesizer
- Guard against close-code 4006 by tracking the gateway session a voice token
belongs to and refusing to connect with stale credentials
- Lock-guard the streaming TTS converter; O(1) frame draining in playback
- Clean voice teardown on stopBot and /announce rejoin support

DAVE diagnostics:
- Surface every DAVE handshake/transition/roster event to the in-app voice
diagnostics log (previously OSLog-only), including the version-0 downgrade path
- Keep spoken message content out of the diagnostics log
