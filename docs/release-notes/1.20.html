Security Audit Hardening, Audit Logging, and Welcome Flow v2

Version 1.20

A comprehensive security and management overhaul following a detailed audit of the Admin Web Server. This release hardens the bot's web interface against common attack vectors and introduces new tools for tracking administrative activity.

Security Hardening:
- Enforce mandatory Discord MFA for Web UI login.
- Implement failure-window rate limiting and lockout for login and bearer token attempts.
- Migrate session persistence from UserDefaults to secure macOS Keychain storage.
- Reduce session TTL to 24 hours and implement User-Agent session binding.
- Replace static media tokens with short-lived, HMAC-signed access tokens.
- Implement constant-time string comparisons for all credential and token validations.
- Enforce modern security headers (HSTS, X-Frame-Options, X-Content-Type-Options).

Features & Management:
- Implement a structured Audit Log for tracking authentication, configuration, and moderation events.
- Redesign the Activity Log with unified filtering for Audit, Gateway, System, and Mesh categories.
- Complete refactor of Welcome Flow (v2) with rich embed support and invite tracking.
- Refactor WikiBridge into the more versatile Lookup service.
- Add support for local timezones and dynamic timestamp formatting in command processing.

Core Improvements:
- Full migration to Swift 6.0 with strict concurrency checking.
- Refresh onboarding animations and unify dashboard metric grids.
- Improved cluster coordination and connection diagnostics.
