SwiftBot 1.21.2

Version 1.21.2

A focused update for the Announcer: it no longer falls silent mid-session, it can follow a member into a Go Live stream, and you get direct control over reading a voice channel's own chat.

🔊Reliable Speech

The Announcer now keeps talking through long, quiet stretches.

  • Fixed Random Silence: Resolved an issue where the Announcer would light up as "speaking" in Discord but play no audio after a gap between messages.
  • Warm Voice Stream: The connection is now kept alive between announcements with a paced silence keepalive, so the next message is heard immediately instead of being dropped.
  • Steady Timing: Voice packet timing now stays aligned with real time across idle periods, which was the root cause of the drop-outs.

🎥Go Live Auto-Join

SwiftBot can hop into a channel the moment someone starts streaming.

  • Auto-Join on Stream: Added a per-channel rule to join automatically when a member starts a Go Live stream, alongside the existing auto-join when someone enters.
  • Spoken Stream Intro: An optional announcement on stream join — SwiftBot can say "I'll read notifications during the stream." as it connects.
  • Familiar Controls: Stream joins use the same stay-connected options — fixed duration or until the channel empties — as a regular auto-join.

💬Reading the Stream Chat

Clearer control over the voice channel's built-in text chat.

  • Read Voice Channel Chat: Added a toggle to read the voice channel's own Text-in-Voice chat — the chat viewers see beside a Go Live stream.
  • Zero-Config Stream Reading: A stream auto-join can now run with no separate text channel set up; the bot joins and reads the stream chat out of the box.
  • Focused Sources: Turn it off to limit the Announcer to only the specific text channels you've added.
