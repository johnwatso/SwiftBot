Support multiple watched text channels and tests

Allow tracking multiple text channels (comma-separated) for the voice announcer and ensure the active voice channel is included as a watched source. Refactor TextChannelAnnouncer to use a Set of watched channel IDs and add setWatchedChannels API; update AppModel (AdminWeb and Voice) to persist and initialize multiple channel IDs, resolve channel names for logs, and forward messages from either watched text channels or the active voice chat. Update VoiceView to display multiple channel names, add tests (TextChannelAnnouncerTests) that cover multi-channel handling and forwarding, and register the new test file in the Xcode project. Also adjust a few file metadata entries in project.pbxproj and bump MARKETING_VERSION to 1.21.1.
