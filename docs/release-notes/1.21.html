SwiftBot 1.21

Version 1.21 · Changes since SwiftBot 1.20

SwiftBot 1.21 builds on the 1.20 security and clustering release with SwiftMiner companion sign-in, tunnel hostname sharing, better app presence controls, stronger failover pairing, and fixes carried forward from the 1.20.1 maintenance line.

🔗SwiftMiner Companion

SwiftBot and SwiftMiner now cooperate more cleanly when sharing Discord sign-in and hosted dashboards.

  • Added a SwiftMiner companion OAuth entry point that validates registered return hosts and redirects with a short-lived signed identity assertion.
  • Added signed registration for SwiftMiner dashboard hostnames on SwiftBot-managed Cloudflare tunnels.
  • Persisted companion hostname rules so tunnel reconfiguration keeps SwiftBot and SwiftMiner ingress routes together.
  • Re-checks companion DNS records during tunnel setup reruns, reducing manual Cloudflare cleanup after changes.
  • Requests Discord's guilds scope during OAuth and records whether the signed-in user belongs to a connected SwiftBot guild.

🖥️App Presence and Settings

The Mac app is easier to keep where you want it: in the Dock, in the menu bar, or both.

  • Added Dock and menu bar presence options, including a SwiftBot menu bar item with quick access to the app, Settings, status, dashboard, updates, and Quit.
  • Added a dedicated menu bar asset so the status item stays visually consistent with the app icon.
  • Made Settings opening use SwiftUI's Settings scene opener for modern macOS behavior.
  • Clarified Failover read-only behavior in General Settings while keeping local recording folders editable on each Mac.

🕸️SwiftMesh Pairing

Failover nodes recover and resync more reliably when they are paired again.

  • Included the Primary leader term in SwiftMesh join codes and registration acknowledgements.
  • Made re-paired Failover nodes adopt the registered Primary term before initial sync, preventing stale local terms from rejecting Primary pushes.
  • Included the live dashboard snapshot in initial resync responses so Failover app views populate immediately after pairing.
  • Added integration coverage for stale-term re-pairing and initial live snapshot resync.

🎙️Voice and Sweep

Voice analytics now better match what happened during the rolling seven-day window.

  • Counts active voice sessions and sessions that overlap the seven-day window instead of only sessions that started inside it.
  • Clips long-running sessions to the rolling window so leaderboard totals are accurate.
  • Ranks voice users by total rolling voice time with deterministic name ordering for ties.
  • Renamed Sweep's pinned voice metric from an average to a seven-day voice leaderboard so the label matches the displayed total time.
  • Added focused tests for active sessions, clipped sessions, and rolling voice totals.

✅1.20.1 Maintenance Fixes

All 1.20.1 fixes are included in 1.21.

  • Improved Patchy validation coverage for malformed or incomplete update data.
  • Removed stale Patchy UI code that no longer matched the current monitoring flow.
  • Hardened certificate handling and added regression coverage for certificate-management behavior.
  • Improved Cloudflare DNS provider behavior for tunnel and hostname setup.
  • Adjusted Cloudflare tunnel client behavior for the current Web UI preferences flow.

🚢Release and Dashboard Polish

The surrounding release and companion-dashboard experience received small but important refinements.

  • Updated release notes styling and metadata for the 1.20 and 1.20.1 release pages.
  • Simplified SwiftMiner DMs so actions live on the SwiftMiner web dashboard instead of inside Discord buttons and modals.
  • Added a one-time web dashboard availability announcement and dashboard footer links in SwiftMiner notifications.
  • Kept legacy button responses working for older SwiftMiner messages while moving new messages to the dashboard-first flow.
