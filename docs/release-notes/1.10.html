SwiftBot 1.10

Version 1.10 · Build 1100000

The 1.10 release closes out the 1.9 line with two big additions — first-class SwiftMiner integration and a brand-new analytics dashboard — alongside a round of UI polish and repo housekeeping.

⛓️SwiftMiner Integration

SwiftBot can now pair with SwiftMiner directly, removing the manual wiring that previous releases required.

  • Adds a SwiftMiner client, supporting UI, and inbound webhook handling so miners can be created and managed end-to-end through SwiftBot.
  • Adds a built-in pairing server and a Discord linking flow that walks users through connecting an existing SwiftMiner instance.
  • Adds an automatic SwiftMiner setup DM when a new pairing is initiated, with documentation links for first-time users.
  • Polishes the SwiftMiner pairing experience and the connected-miner dashboard for clearer status and fewer dead-ends.

📊Analytics Dashboard

A native and web analytics dashboard now ships with SwiftBot for the first time.

  • Adds an Operational Pulse summary covering voice sessions, total voice time, the most active day, and the top user.
  • Adds an Activity Overview line chart for the last seven days, plus a Live Operations feed of recent runtime events.
  • Adds System Activity and Bot Health panels covering command throughput, failed actions, gateway events, queue depth, concurrent tasks, and memory.
  • Adds Insights and Top Users panels generated from live app state.
  • Mirrors the same dashboard in the admin web UI so analytics are available away from the Mac.
  • Switches the activity line chart to monotone interpolation and adds y-axis headroom so the curve and peak markers stay inside the panel.

🪟UI and UX Polish

Smaller refinements across the app surface.

  • Polishes the SwiftBot sidebar layout and grouping.
  • Refreshes the admin web UI to align with the native dashboard.
  • Updates the in-app preview imagery to reflect the new analytics view.

🧱Reliability and Internals

Continued runtime hardening on top of the 1.9 refactor.

  • Hardens Patchy AMD release-note fetch handling against transient failures and malformed responses.
  • Tightens media export and library indexer paths for steadier long-running exports.
  • Cleans up app-owned security components and clarifies their boundaries with the rest of the runtime.

🧹Repo and Project Housekeeping

Non-code changes that ship alongside this release.

  • Switches the project license to MIT and refreshes the README style.
  • Adds a security policy and removes the standalone vulnerability reporting section in favour of it.
