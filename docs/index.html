Get SwiftBot running in minutes.

Create the Discord application, paste the token, choose what SwiftBot should handle, and add failover when you need it.

01

Create your Discord app

Make a bot in the Discord Developer Portal, copy the token, and enable the required gateway intents.

02

Launch SwiftBot

Paste the token during onboarding. SwiftBot validates it, stores it in Keychain, and builds the invite URL.

03

Configure workflows

Set commands, automation rules, AI providers, Patchy sources, Sweep cleanup, and Web UI access.

04

Add failover when ready

Pair standby or worker nodes with SwiftMesh join codes and test the handover path before you need it.

Answers for common Discord bot searches.

SwiftBot is built for server owners who want a practical Discord bot they can run themselves from a Mac.

Yes. SwiftBot runs locally on Apple Silicon Macs, connects through your Discord application, stores the bot token in Keychain, and operates in your server like a normal Discord bot.

SwiftBot gives you configurable rules for message events, joins, roles, DMs, moderation actions, and channel routing, so routine server workflows can be managed from the dashboard.

SwiftBot supports AI replies and WikiBridge lookup commands, which helps communities answer common setup, support, and knowledge-base questions without repeating the same response manually.

SwiftBot is designed around managing one primary Discord server. It can be invited to more than one server, but the dashboard and workflows are clearest when one server is the main home.

The Admin Web UI can stay local or be published through SwiftBot's Cloudflare Tunnel flow, giving trusted admins browser access when the Mac is running elsewhere.

SwiftBot is free and open source under the MIT license. It is actively developed and updated through regular releases on GitHub.
