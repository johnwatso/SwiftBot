One control surface for the whole bot.

SwiftBot keeps day-to-day bot operations in one native app instead of scattered config files, terminal logs, and browser tabs.

Native dashboard

Monitor gateway status, commands, logs, diagnostics, permissions, latency, and bot health from a SwiftUI macOS interface.

Automation rules

Build rules for messages, joins, voice events, moderation, AI replies, role actions, DMs, and channel routing.

AI and knowledge

Route AI replies through Apple Intelligence, Ollama, or OpenAI, and use WikiBridge-backed lookup commands.

SwiftMesh failover

Run primary and standby nodes, sync live state, rehearse handovers, and keep Discord output gated to the active leader.

Patchy alerts

Track AMD, NVIDIA, Intel, Steam, GitHub, and other update sources, then send clean Discord notifications.

Admin Web UI

Manage SwiftBot from a browser when needed, with Discord OAuth and optional Cloudflare Tunnel setup.

Native app first, browser access when useful.

SwiftBot is built for macOS, but the Web UI makes remote checks and admin tasks easier when the bot is running elsewhere.

macOS dashboard

Operate the bot without leaving the app.

Use the sidebar to move between runtime status, commands, automations, Patchy, AI providers, diagnostics, SwiftMesh, logs, analytics, and settings.

Web admin

Check in from another device.

The built-in Web UI mirrors key management surfaces and can stay local-only or be published through SwiftBot's Cloudflare Tunnel flow.

Setup is guided, not mysterious.

Bring your own Discord application and SwiftBot walks the rest of the operational pieces into place.

01

Create your Discord app

Make a bot in the Discord Developer Portal, copy the token, and enable the required gateway intents.

02

Launch SwiftBot

Paste the token during onboarding. SwiftBot validates it, stores it in Keychain, and builds the invite URL.

03

Configure workflows

Set commands, automation rules, AI providers, Patchy sources, Sweep cleanup, and Web UI access.

04

Add failover when ready

Pair standby or worker nodes with SwiftMesh join codes and test the handover path before you need it.
