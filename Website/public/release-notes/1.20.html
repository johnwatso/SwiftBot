SwiftBot 1.20

Version 1.20 · Build 2026060313

The biggest SwiftBot update yet, with a security overhaul, stronger SwiftMesh clustering, the new Sweep subsystem, richer automations, and the migration to Swift 6.

🛡️Security and Auditing

Admin access and web sessions are now much more tightly controlled.

  • Role-Based Access Control: Added Viewer and Admin roles, with state-changing Web UI operations gated behind admin permissions.
  • Payload Validation: Added field-level validation for automations, Sweep policies, and webhook payloads so configuration stays inside safe operating bounds.
  • HTTPS Enforcement: Added Require HTTPS mode and cleartext blocking for routable addresses, while keeping local loopback access practical.
  • Hardened Login: Added PKCE for Discord OAuth, mandatory Discord MFA checks, session binding, and IP-based rate limiting.
  • Audit Trail: Added persistent audit logging for authentication, configuration changes, and moderation activity.

🕸️SwiftMesh and Clustering

Multi-node SwiftBot setups became safer and easier to understand.

  • Encrypted Mesh Traffic: SwiftMesh sync traffic now uses AES-GCM encryption keyed from the shared secret.
  • Failover Hardening: Added split-brain protection, confirmed-retry behavior, cursor preservation, and auto-reclaim for configured Primary nodes.
  • Cluster Visibility: Added a compact cluster map, node icon overrides, live latency tracking, healthy-node summaries, and better diagnostic status.
  • Handover Testing: Added tools for testing handover behavior before relying on a cluster during a real outage.

🧹Sweep and Automations

Server maintenance and member workflows got more capable.

  • Sweep: Added a subsystem for finding missing permissions, inactive roles, and other server health issues.
  • Dry-Run Previews: Added previews so Sweep policies and automation rules can be checked before they make changes.
  • Welcome Flow: Refactored member join and leave handling with rich embeds, invite tracking, and account-age gating.
  • Lookup Expansion: Expanded Lookup and Patchy source handling with GitHub, Steam, and improved wiki behavior.

✨UI and Experience

The native app and Web UI were brought into a more consistent shape.

  • Visual Refresh: Updated native views and the admin Web UI with clearer hierarchy, more consistent controls, and better dashboard density.
  • Activity View: Merged command and system logs into a unified Activity view with smarter filtering for Gateway, Audit, and Mesh events.
  • Login Polish: Refined the Web UI login and MFA screens while preserving performance on heavier pages.
  • Automation Editing: Added variable autocomplete and improved preview placement for workflow configuration.

⚙️Core Infrastructure

The app foundation moved forward for the next release cycle.

  • Swift 6: Migrated the codebase to Swift 6 with stricter concurrency checking.
  • Voice Runtime: Updated Opus, UDP, TTS, and Discord voice handling for more reliable playback.
  • Recording Playback: Improved recording streaming and playback behavior.
  • Timezone Support: Added local timezone and timestamp formatting support in command responses.

📖Documentation

Setup and deployment guidance is easier to find.

  • Added bot setup documentation covering tokens, intents, OAuth, and invite links.
  • Added Web UI setup documentation, including Cloudflare Tunnel guidance.
  • Updated agent and architecture notes for the new release workflow.
