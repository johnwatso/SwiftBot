SwiftBot 1.22.2

Version 1.22.2 · Build 2026070311

A focused Announcer update: quicker rejoins, clearer diagnostics when messages are skipped, a tighter voice setup card, and DAVE identity fixes for encrypted voice sessions.

⚡Faster Voice Recovery

Reconnects now wait for Discord's real leave acknowledgement instead of burning a fixed delay.

  • Quicker manual reconnects: The Announcer proceeds as soon as Discord confirms the bot left the old voice session, with a short timeout only as a fallback.
  • More responsive /announce rejoin: Slash-command rejoins use the same acknowledgement path so Discord is less likely to ignore the new join as a duplicate voice state.
  • Faster first recovery attempt: Automatic recovery now retries after 750 ms for the first attempt, then backs off to 5 seconds and 15 seconds if the connection is still unstable.
  • Cleaner startup auto-connect: Auto-connect arms once after gateway READY and fires when the channel cache actually contains the configured voice channel, instead of polling on a timer.
  • Safer manual disconnects: A deliberate disconnect disarms startup auto-connect, so a later channel-cache sync cannot quietly rejoin the Announcer.

🔊Announcer Reads

It is easier to tell why a message did or did not get spoken.

  • Skipped-message explanations: The voice log now reports when a watched message is skipped because it is too long, mostly emoji, or has no readable text after links and attachments are ignored.
  • Long-message behavior is clearer: Over-cap messages are skipped with a concrete reason unless "Shorten long messages" is enabled, in which case a shortened version is queued.
  • First intro plays sooner: Manual join introductions are delivered as soon as the voice pipeline is live instead of waiting on a repeated polling loop.
  • Live timing is logged: SwiftBot now records how long it took for Announcer voice to become live after a join request.
  • Permission checks are smarter: Voice-channel REST preflight is cached for channels that already passed during this app run, then refreshed after failed connects so permission changes can still be caught.

🎛️Announcer Interface

The Voice tab is denser, easier to scan, and less jumpy.

  • Tidier voice-channel cards: Status, enable, preview, edit, and overflow actions now sit in a compact header row, with Feeds and Behaviour grouped underneath.
  • Matched dashboard cards: Current State and Recent Activity now share an equal-height row so one card does not tower over the other.
  • Better status symbols: Connection and queue states use clearer SF Symbols such as checkmark.circle.dotted and clock.badge.
  • Static Current State title icon: The Current State card title now uses bolt.badge.checkmark.fill while the live state inside the card remains dynamic.
  • Cleaner Announcer identity: The sidebar and page header now use person.wave.2.fill, better matching the spoken-announcement feature.

🔐DAVE Voice Encryption

Encrypted voice sessions keep a stable MLS identity and report transitions more accurately.

  • libdave-swift 1.3.1: SwiftBot now uses the newer DAVE framework build that supports a persisted key store for authenticated sessions.
  • Stable bot identity: DAVE sessions are configured with an auth session id based on the bot user, so MLS signature identity can persist across voice sessions instead of starting from scratch each time.
  • Safer transition diagnostics: Deferred DAVE execute-transition logs now report applied transition counts instead of stale epoch wording.
  • Regression coverage: The test suite now verifies that an authenticated DAVE coordinator can produce a non-empty MLS key package.

🚢Release Polish

Project metadata and tests were refreshed for the 1.22.2 build.

  • Updated MARKETING_VERSION to 1.22.2 and CURRENT_PROJECT_VERSION to 2026070311.

  • Added tests for skipped long-message logging, shortened long-message reads, voice recovery backoff, and authenticated DAVE key-package generation.
