SwiftBot 1.21

Version 1.21

A clustering release that lets Failover (standby) nodes safely forward configuration edits to the Primary instead of writing replicated config locally, so multi-node setups stay consistent.

🕸️SwiftMesh Failover

Standby nodes can now edit configuration through the Primary rather than writing replicated config locally.

  • Config authority: Failover GUI surfaces forward typed config edits to the Primary via mutation envelopes; the Primary applies and acknowledges each change.
  • Safe by design: Mutations are idempotent (deduplicated by client mutation ID), failures are surfaced explicitly, and an immediate reconciliation pull follows each change.
  • Broad coverage: Patchy, Automations, Commands, Welcome Flow, Voice, Lookup, and Sweep edits all route through the new authority path.
  • Sweep sync: A dedicated Sweep policy sync endpoint handles Sweep state, which lives outside BotSettings.

⚙️Testing

New coverage for the failover authority path.

  • Added MeshConfigMutationTests for leader-only handling, mutation acknowledgement, Codable round-tripping, and duplicate mutation-ID handling.
